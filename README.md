# 🌡️ Melbourne Urban Heat Island Explorer
### An Open-Source Web GIS Application

> An interactive, browser-based web map exploring the spatial relationship between Land Surface Temperature, building typology, and tree canopy coverage across Melbourne, Australia.

**Live map:** [https://YOUR_USERNAME.github.io/melbourne-heat-explorer](https://YOUR_USERNAME.github.io/melbourne-heat-explorer)

---

## 📌 Overview

Urban Heat Islands (UHIs) occur when cities absorb and re-emit solar radiation through concrete, asphalt, and built surfaces — creating pockets of intense heat. In Melbourne, surface temperatures can vary by up to **10°C** across just a few kilometres.

This application visualises **Land Surface Temperature (LST)** derived from Landsat 8/9 satellite imagery across Melbourne City SA2 areas, overlaid with building footprints and live tree canopy data. It was built as part of a full local-to-cloud GIS pipeline:

```
PostGIS → QGIS → QGIS2Web → VS Code → GitHub Pages
```

---

## 🗺️ What the Map Shows

| Layer | Description |
|---|---|
| 🌡️ **LST Zones** | Five-class land surface temperature choropleth derived from Landsat 8/9 Band 10. Classified using Natural Breaks (Jenks). |
| 🏢 **Building Footprints** | Melbourne City building polygons attributed by roof type (flat, hip, gable, pyramid) and average height. |
| 🌳 **Tree Canopy** | 1000+ trees fetched live from the City of Melbourne Urban Forest API, coloured by age class. |

---

## ✨ Features

- **LST Temperature Filter** — drag a slider to isolate temperature zones above a chosen threshold
- **Temperature Distribution Histogram** — bar chart showing SA2 area count per temperature class, matching the map's Spectral colour ramp
- **Live Tree API** — tree canopy data fetched at runtime from the Victorian Government CKAN DataStore API; not a static file
- **Tree Age Colouring** — young, mature, and over-mature trees rendered in distinct colours
- **Collapsible About Panel** — study purpose, aim, and dataset sources
- **Address Search** — Nominatim OpenStreetMap geocoder for locating suburbs and streets
- **GPS Locator** — centres the map on the user's current position
- **Dynamic Scale Bar** — updates automatically with zoom level
- **Attribution** — full data source credits at bottom centre

## 🛠️ Technology Stack

| Tool | Role |
|---|---|
| **PostGIS / PostgreSQL** | Spatial database — source of truth for LST and building data |
| **QGIS 3.x** | Desktop GIS — styling, CRS verification, layer configuration |
| **QGIS2Web (Leaflet)** | Exports styled project to self-contained HTML/CSS/JS/GeoJSON |
| **Leaflet.js** | Web mapping library — powers the interactive map |
| **Chart.js 4.4.0** | JavaScript charting library — renders the LST histogram |
| **VS Code** | Code editor — used to extend the exported HTML with custom tools |
| **GitHub Pages** | Static site hosting — serves the application as a live public URL |

## 🌐 Data Sources

| Dataset | Source | Access |
|---|---|---|
| Land Surface Temperature | Landsat 8/9 Band 10 — USGS/NASA, processed via Google Earth Engine | Static GeoJSON in `/data/` |
| Building Footprints | City of Melbourne Open Data | Static GeoJSON in `/data/` |
| Tree Canopy — Urban Forest | City of Melbourne Urban Forest, via Victorian Government DataVic | Live API at runtime |
| Basemap | CartoDB Positron (retina) | Tile layer via CDN |

**Tree Canopy API endpoint:**
```
https://discover.data.vic.gov.au/api/3/action/datastore_search
?resource_id=0f2a2180-2be0-5a58-a270-7538c35259e6&limit=1000
```

**Dataset landing page:**
[https://discover.data.vic.gov.au/dataset/trees-with-species-and-dimensions-urban-forest](https://discover.data.vic.gov.au/dataset/trees-with-species-and-dimensions-urban-forest)

> The tree data is fetched live via HTTP request each time the map loads. The underlying dataset is periodically updated by the City of Melbourne — not a real-time sensor feed. The LST and building layers are frozen GeoJSON files exported from PostGIS and do not update automatically.

## 🌡️ LST Classification

Temperature zones were classified using Natural Breaks (Jenks) on the `landsat8_9` field:

| Class | Range | Label | Colour |
|---|---|---|---|
| 1 | 17.7 – 19.3°C | Low Temperature Zone | 🔵 Blue |
| 2 | 19.3 – 21.5°C | Sub-Low Temperature Zone | 🟢 Green |
| 3 | 21.5 – 23.3°C | Medium Temperature Zone | 🟡 Yellow |
| 4 | 23.3 – 26.0°C | Sub-High Temperature Zone | 🟠 Orange |
| 5 | 26.0 – 27.2°C | High Temperature Zone | 🔴 Red |

## 🚀 Running Locally

No server or build tools are required. The application is a static HTML folder.

1. Clone the repository:
```bash
git clone https://github.com/YOUR_USERNAME/melbourne-heat-explorer.git
```

2. Open the folder in VS Code:
```
File → Open Folder → melbourne-heat-explorer
```

3. Open `index.html` directly in your browser, or use the VS Code Live Server extension for auto-refresh.

> An internet connection is required to load the CartoDB basemap tiles, Chart.js CDN, and the live City of Melbourne Tree Canopy API.
